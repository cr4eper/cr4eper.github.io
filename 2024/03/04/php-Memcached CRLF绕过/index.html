php-Memcached CRLF绕过

基础概念

Memcached介绍

Memcached是一个php中用于数据缓存的工具，能帮助用户有效的存储和检索的数据；可以将Memcached想象成一个高速的存储空间，类似于一个大型的缓存存储库。它可以存储各种类型的数据，例如数据库查询结果、API响应、计算结果等。他的工作原理是这样的：当我们的应用程序需要某些数据时，首先会检查Memcached中是否已经缓存了该数据。如果存在缓存，应用程序可以直接从Memcached中获取数据，而不需要执行一些耗时的操作，比如从数据库中读取数据。这样可以大大提高性能，减少响应时间。

在Memcache中，是通过键值对的形式对数据进行存储的：

键(key): 键名不可重复, 一般为字符串, 最大不可超过 128 个字节。
值(value): 值的格式可为字符串、数值、数组、对象、布尔、二进制、NULL

CRLF注入漏洞

在HTTP报文中，状态行和首部中的每行以CRLF结束，首部与主体之间由一空行分隔。而CRLF漏洞的产生则是因为Web应用没有对用户输入做严格验证，导致攻击者可以输入一些恶意字符串，攻击者一旦成功注入CRLF，就会在响应中输入，所以也称为HTTP响应拆分漏洞。

漏洞环境与命令

环境

该漏洞存在于2.20版本及其以下的Memcached中，

命令

常用的有set命令和get命令

set命令

Memcached的set命令用于将value存储在指定的key中，如果key已存在则会更新key值。

基本语法如下：

set key flags exptime bytes [noreply] 
value
  • key：键值 key-value 结构中的 key，用于查找缓存值。
  • flags：可以包括键值对的整型参数，客户机使用它存储关于键值对的额外信息 。
  • exptime：在缓存中保存键值对的时间长度（以秒为单位，0 表示永远）。
  • bytes：在缓存中存储的字节数。
  • noreply（可选）： 该参数告知服务器不需要返回数据。
  • value：存储的值（始终位于第二行）（可直接理解为key-value结构中的value）。

在上面的执行的代码中key的值是game，flag就是0，(exptime)过期时间为100，（bytes）字节长度为8，value的值为yuanshen，在设置成功后会显示STORED。

get命令

Memcached的get命令获取存储在key中的value，如果key不存在则会返回为空。

基本语法如下：

get key
或
get key1 key2 key3

key：键值 key-value 结构中的 key，用于查找缓存值。

可以看到我们获取到实例game，并且输出END表示结束。

漏洞注入

假设我们存在下面这样的一段代码：

<?php
    $server->addServer('localhos',11211);
    $token = $GET['token'];
    $server->set("wolf","poc");
    echo "[token] = ";
    var_dump($server->get("$token"));
    echo "[wolf] = ";
    var_dump($server->get("wolf"));
?>

假设部署到了网站上，便可以利用get位置输入的缓存信息，可以利用其对网站进行注入：

http://127.0.0.1/index.php?token=monkey%00%0D%0Aset%20allbmonkey%200%20500%203%0D%0Acat

//等同于
get monkey
set allbmonkey 0 500 3
cat

例题

2023强网杯Thinkshop[ping]

题目中给了源码，我们先看源码，下面是源码的目录结构：

我们先看Admin.php，源码中包含登陆的函数，具体如下：

public function do_login()
    {

        $username = input('post.username');
        $password = input('post.password');
        

        if (empty($username) || empty($password)) {
            $this->error('用户名或密码不能为空');
        }
        if(strlen($password) > 100)
        {
            $this->error('用户名或密码错误');
        }

        // 使用md5对输入的密码进行加密
        $encryptedPassword = md5($password);

        // 设置缓存键和有效期
        $Key = ["Login" , $username];
        $Expire = 600; // 缓存有效期为10分钟 (600秒)

        // 尝试从缓存中获取数据
        $adminData = Db::table('admin')
            ->cache(true, $Expire)
            ->find($username);

        if ($adminData && $adminData['password'] === $encryptedPassword) {
            // 登录成功，设置session
            session('admin', $adminData['username']);


            $this->success($username.'登录成功', 'index/admin/goods_edit');
        } else {
            $this->error('用户名或密码错误');
        }
    }

其中的关键在于下面的代码：

$adminData = Db::table('admin')
            ->cache(true, $Expire)
            ->find($username);

这段代码的意思是：

  • Db::table(‘admin’)：这行代码指定要从名为 “admin” 的数据库表进行查询操作。Db 是一个数据库操作类（可能是一个自定义的类），table(‘admin’) 方法用于指定要查询的表名。

  • ->cache(true, $Expire)：这行代码启用了查询结果的缓存功能。cache(true, $Expire) 方法调用了一个名为 cache 的函数，第一个参数为 true，表示启用缓存，第二个参数 $Expire 是一个变量，用于指定缓存的有效期（以秒为单位）。

  • ->find($username)：这行代码执行了查询操作，查找具有特定用户名 $username 的记录。find($username) 方法调用了一个名为 find 的函数，该函数执行了根据用户名进行查询的操作，并返回查询结果。

简单来说就是在指定有效期中，如果我们进行了多次查询，就比如说我们多次登陆，他就不会从数据库中查询，而是从缓存中查询数据表admin的值。

再看find函数的源码：

$options['limit'] = 1;
$result = false;

// 检查是否启用了查询缓存
if (empty($options['fetch_sql']) && !empty($options['cache'])) {
    $cache = $options['cache'];

    // 判断缓存键的生成方式
    if (true === $cache['key'] && !is_null($data) && !is_array($data)) {
        // 生成缓存键
        $key = 'think:' . $this->connection->getConfig('database') . '.' . (is_array($options['table']) ? key($options['table']) : $options['table']) . '|' . $data;
    } elseif (is_string($cache['key'])) {
        $key = $cache['key'];
    } elseif (!isset($key)) {
        // 生成默认缓存键
        $key = md5($this->connection->getConfig('database') . '.' . serialize($options) . serialize($this->bind));
    }

    // 从缓存中获取结果
    $result = Cache::get($key);
}

可以发现key的形式是$key='think:database_name.table_name|data'，也就是$key='think:shop.admin|admin'
