java安全——反射

反射

静态语言与动态语言区别：动态语言在修改代码后会对运行中的代码产生影响，而静态不会。

Reflection (反射)是Java被视为动态语言的关键,反射机制允许程序在执行期借助于Reflection API取得任何类的内部信息，并能直接操作任意对象的内部属性及方法。如下：

Class c = Class.forName("java.long.String")

加载完类之后，在堆内存的方法区中就产生了一个Class类型的对象(一个类只有-个Class对象) ，这个对象就包含了完整的类的结构信息。我们可以通过这个对象看到类的结构。这个对象就像一 面镜子，透过这个镜子看到类的结构，所以，我们形象的称之为:反射

反射基本概念：

在Java中，反射（Reflection）是指在运行时动态地获取、检查和操作类、对象、方法和属性的能力。反射机制允许程序在运行时获取类的信息、调用方法、访问属性，而不需要在编译时知道这些类的具体信息。

通过反射，可以在运行时获取类的相关信息，如类的名称、继承关系、接口实现、构造函数、方法和属性等。并且可以通过反射创建类的实例、调用类的方法、获取和设置类的属性的值，即使这些类在编写代码时并不可见或可知。

Java中的反射机制主要通过java.lang.reflect包提供的类和接口来实现，其中最重要的类包括Class、Constructor、Method和Field等。

  • Class类：用于表示类的实例，在运行时可以通过它获取类的各种信息，如类的名称、修饰符、父类、接口、构造函数、方法和字段等。
  • Constructor类：用于表示类的构造函数，可以通过它创建类的实例。
  • Method类：用于表示类的方法，可以通过它调用类的方法。
  • Field类：用于表示类的字段（属性），可以通过它获取和设置类的属性的值。

Class类

这个个类是表示其他类的共同点的总和，简单来说他就是类的抽象类，然后调用其中的方法

什么时候会发生类的初始化：

类的主动引用(- -定会发生类的初始化)

  • 当虚拟机启动，先初始化main方法所在的类
  • new- -个类的对象
  • 调用类的静态成员(除了final常量)和静态方法
  • 使用java.lang.reflect包的方法对类进行反 射调用
  • 当初始化- 个类，如果其父类没有被初始化，则先会初始化它的父类

类的被动引用(不会发生类的初始化)

  • 当访问- -个静态域时，只有真正声明这个域的类才会被初始化。如:当通过子类弓|用父类的静态变量
  • 致子类初始化
  • 通过数组定义类引用，不会触发此类的初始化
  • 引用常量不会触发此类的初始化(常量在链接阶段就存入调用类的常量池中了)

类加载器的作用

类加载的作用:将class文件字节码内容加载到内存中，并将这些静态数据转换成方法区的运行时数据结构，然后在堆中生成一个代表这个类的java.lang.Class对象，作为方法区中类数据的访问入口。类缓存标准的JavaSE类加载器可以按要求查找类,但一旦某个类被加载到类加载器中，它将维持加载(缓存)一段时间。不过JVM垃圾回收机制可以回收这些Class对象

类的加载过程：

获取正在运行类中的信息

Class c1 = Class.forName("reflection.cheshi1.Uaer");
       System.out.println(c1.getName());//获取包名、类名
       System.out.println(c1.getCanonicalName());//获取类名


       //获取类的属性
       Field[] fields = c1.getField();//只能获取public属性的变量
       fields = c1.getDeclaredFields();//获取全部属性的变量
       Field name = c1.getDeclaredField("name");


       //获得类方法
       Method[] fields1 = c1.getMethods();//获取当前类的所有public方法
       fields1 = c1.getDeclaredMethods();//获取当前类的所有属性的方法


       //获得指定类的方法
       Method getName = c1.getMethod("getName",null);//获取getName方法


       //获取指定的构造器
       Constructor[] ch2 = c1.getConstructors();


       //获得指定的的构造器
       Constructor ch3 = c1.getDeclaredConstructor(String.class,int.class,String.class);
